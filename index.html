今天的leetcode的题目其实挺简单的，就是从长字符串里找短字符串，但有一点还是让我觉得值得注意的：就是速度和内存。资源的消耗在我们这个等级其实并不是很必要（to be honestly并且承认自己很菜）。

这道题的最简单（时间上）方法就是默认return True，只判别具体错误实例，虽然只有几毫秒的差别，但人家就是能想到这样提速。我觉得还是很厉害的。

之前重写预测模型时我就在想怎么把读数据写的快一点，但这个数据量提速没有什么显著效果，且根本没有达到硬件限制，所以最后也不了了之。在模型上怎么提速？
这是一个问题，参数的优化就是提速吗？写的过程中很多加载和判断的地方其实很占内存，在服务器上影响不大，但略微的优化或许就能在自己的pc上运行完成了。这些想法怎么实现？现在还真没啥心思搞这个了，真是遗憾。

因为实验室的webserver是Linux系统，而我日常使用win，初次接触还是有很多命令要学习，确实很令我头疼。

在这里，记录一下一个初学者的commad:

ls 列出当前目录下的文件 
    ls -l 是以列表形式的展示，简写： ll
    ls在windows的powershell中也支持，默认以列表展示（仅支持ls）

pwd 当前目录

cd target_path 与windows一致，切换路径

cat filename 查看文件的全部内容
    head -n filename 查看文件头部，-n可以选择前n行查看
    tail -n filename 查看尾部，-n查看最后n行

top 查询当前进程，一般我用来看是否跑完了模型，之前想写一个邮件提醒的，但因为不会配置放弃了，还是回到了脑动预估时间、手动查看进程的原始人年代

ps -aux  列出全部进程
    a 全部进程 u 用户 ux 用户全部进程 

grep target filename 在文件中查询特定字段

sed 从文件中删除特定条件
    这里以我常用的一条命令为例：
    sed -i '/Warning/d' nohup.out 
    /d 是删除整行，因为nohup.out中经常带有大量的warning，但warning又不是error，我不在意，只想知道跑到哪一步出问题，如此处理就只剩下我关心的内容了。

nohup 不挂起
    因为目前模型普遍需要一个晚上的时间，nohup可以将任务后台不挂起解放我的笔记本回去打游戏！工具果然是摸鱼必备。

lv1

建模只是数据处理和脚本运行吗？

是。把到手的鱼毒数据计算一下pLD50，输出一个符合接口要求的标准格式，上传服务器，nohup运行脚本。似乎这套流程下来就可以打开手机摸鱼了。

不是。从毕设数据收集开始，我就遇上了很多困难。文献SI没有提供数据要向作者索要，不好意思，人家根本不理你。有SI数据的，下载回来，但数据有标签不明、提供自定义id没有SMILES、说某个数据库中收集到但数据库已经迁移等各种情况，确实很费精力。Dr.Lee说数据才是我们的生命，确有道理。数据库数据也各有难点，DrugBank提供了XML格式的数据，当时有同时做其他事，没时间看这个XML读取，放弃了XML学习。后来借鉴了yzh的脚本，虽然他在这儿接收不到我的感谢。

def DBID_to_SMILES(IDLIST):
    dicttable = {x:[] for x in ['DBID','SMILES']}
    if len(IDLIST) <= 1:
        print('DBID_num Error')
    try: 
        with tqdm(IDLIST,ascii=True) as t:
            for DBID in t:
                dicttable['DBID'].append(DBID)
                try:
                    r = requests.get(
                        url = 'https://www.drugbank.ca/structures/small_molecule_drugs/{}.smiles'.format(DBID)
                        )
                    dicttable['SMILES'].append(r.text)
                except Exception:
                    dicttable['SMILES'].append('not found')
    except KeyboardInterrupt:
        t.close()
        raise
    t.close()
return pd.DataFrame(dicttable)

实际上，这个简单小脚本依赖于DrugBank网站提供的API，小巧有趣，大概也是python吸引我的地方。

评价函数的探讨

introduction

由于目前在做的模型涉及了多标签的评价方法，这还是我头一次接触AUC、ACC、SE、SP以外的评价函数，特地拿它出来提一提。

涉及的multilabel评价指标为：Subset Accuracy、Hamming Loss、Jaccard Similarity Coefficient和micro AUC。SubACC表示预测标签集和真实标签集完全相同的比例，HL表示n*k次预测中预测错误的比例，JSC表示预测标签集与真实标签集的相似度。

轮子

一开始上来直接对着公式就要造轮子，造到一半突然对HL公式中‘对称差异’产生困惑，再查找时突然后发现sklearn就有这个轮子，sklean.metrics里面包含了各种常用的函数。这里要反思，一直用师兄的脚本，所以对sklearn没有什么印象。
